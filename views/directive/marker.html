The marker directive is used to add a google.maps.Marker object to an existing map.

Usage

Parameters

Required properties are in red.

Param Type Details
idKey expression New in v1.2.0 REQUIRED!!

This was added to avoid unnecessary re-draws on the map.

Expression to evaluate a field as key/id to identify a marker. Normally this is some identifier
which can be used out of your database. Examples: 1,2,3 or "1","2","3", or "one","two","three". Important part, is that they should be unique!
coords expression Expression to evaluate as an object representing the marker location. May be in the form of an object containing latitude and longitude properties, or GeoJSON Point object.
click expression|function Event handler to call when clicking on the marker
options.icon expression

Expression returning the absolute path to an image used as the marker icon

icon is part of options

example

$scope.options = {icon:'url/icon.png'}
control Object New in v1.2.0

{} otherwise undefined / null

If some object is passed (via scope binding) functionality is appended to that object. This allows you to get "control" of some underlying logic of the marker directive..

The following functions are added:

getGMarkers() - Array of all the google.markers associated with this directive

example

$scope.markerControl.getGMarkers();
