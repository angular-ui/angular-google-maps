The markers directive is similar to the marker diective above, but is used when a lot of markers need to be added to a map. It overcomes the high overhead of using the marker directive together with ng-repeat.

Usage

Parameters

Required properties are in red.

Param Type Details
models expression An array of objects defining each marker to add to the map
coords expression The name of the property containing the marker coordinates. Must point to an object having latitude and longitude properties. The special value 'self' can be used to tell the directive that the objects in the array directly contain latitude and longitude properties.
doCluster expression boolean property to tell the directive to use clustering
clusterOptions expression object containing options to pass to the marker clusterer
fit expression boolean property to instruct the map to fit its bounds around the markers
icon expression url to the icon image to use for each marker
click expression event handler executed when clicking a marker. If it resolves to a string and the resultstring is available as a function on the clicked model, this function will be called. If it resolves to a function on the scope it will call this function. optionally injecting the clicked model as $markerModel.
options expression additional options to pass to the marker. See MarkerOptions.
