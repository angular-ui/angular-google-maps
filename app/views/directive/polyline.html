The polyline directive is used to draw an overlay connected to various points on the map. See Polyline API.

Usage

Parameters

Required properties are in red.

Param Type Details
path expression

The coordinates of the points defining the polyline.

May be in the form of an array of objects containing latitude and longitude properties, or New in v1.1 a GeoJSON LineString object.
stroke expression Object to configure the line stroke. Valid properties are color (CSS hex notation), weight (int) and opacity (between 0.0 and 1.0)
clickable expression boolean property to tell the polyline to respond to click events
draggable expression boolean property to make the polyline draggable across the map
editable expression boolean property to make the polyline editable by dragging control points
geodesic expression boolean property to make the polyline follow the curvature of the earth
icons expression TODO
visible expression boolean property which shows or hides the polyline
static expression

New in v1.1 Boolean property which indicates that the polyline coordinates are static. (I.e. will not change).

Declaring a static path will improve the performance of the directive as it does not need to monitor every coordinate in the path for changes during the Angular $digest cycle.

Notes:
  • Declaring a static path implies that it is not editable. As a result, the editable parameter will be ignored.
  • For static paths, the path attribute will watched by reference so the path may be changed in its entirety and the map will update.
  • Changes to the static declaration are ignored after the directive is instantiated.
